# [1.0.3] - 2025-12-29

## Bug修复

### 1. 修复含括号关键词主体全部脱敏时错误触发Fallback

- **修复**:
    - `isAllMasked()` 检查时增加括号内容判断
    - 只有在**主体全部脱敏且没有括号内容**时才触发Fallback
    - 如果有括号内容，即使主体全部脱敏也是合理的（例如：品牌全部脱敏，保留括号信息）

### 2. 修复NLP从括号内提取的元信息覆盖主体元信息

- **修复**:
    - NLP提取时只分析**主体部分**（去除括号后的 `mainBodyKeyword`）
    - 确保主体元信息不被括号内容污染
    - 括号元信息仍通过 `extractBracketMeta()` 单独提取，用于括号内容脱敏

### 3. 修复括号内容错误保留行业信息

- **修复**:
    - 括号智能脱敏时**不再保留行业信息**（即使主体配置 `keep-industry=true`）
    - 括号内容只保留**地域**和**组织类型**，不保留**行业**和**品牌**
    - 理由：括号通常是分支机构或补充信息，行业字段不重要且可能泄露敏感信息

## 改进

### 1. 日志增强：添加括号脱敏规则说明

- **改进**: 在脱敏策略自然语言说明中添加**括号内容脱敏规则**
- **位置**: 策略决策过程部分（第2节）
- **说明内容**:
    - 如果 `keep-bracket=true`：显示"括号内容脱敏规则：完整保留括号内容（keep-bracket=true）"
    - 如果 `keep-bracket=false`：显示"括号内容脱敏规则：智能脱敏括号内容（保留地域、组织类型，不保留行业和品牌）"
- **示例**:
  ```
### 2. 脱敏策略决策过程：原脱敏策略保留行业、组织类型、长度信息，不保留地域、品牌、括号。括号内容脱敏规则：智能脱敏括号内容（保留组织类型，不保留行业和品牌）。
  ```
- **价值**: 明确告知用户括号内容的具体脱敏逻辑，提高透明度
